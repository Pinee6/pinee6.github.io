哈希表

哈希表解决什么问题：快速判断一个元素是否出现在集合里

哈希函数

不同的元素通过哈希函数映射到哈希表的不同索引里

通过取模来保证不超过哈希表索引

如果不同元素映射到同一个索引，那么发生哈希碰撞

哈希碰撞

两种解决方法：拉链法、线性探测法

拉链法

以索引为链表头创建链表，后来的依次排到链表后面

拉链法就是要选择适当的哈希表的大小，这样既不会因为数组空值而浪费大量内存，也不会因为链表太长而在查找上浪费太多时间

线性探测法

保证tableSize>dataSize(总数据量)

依次查找哈希表中的空位

数组Arrays

集合Set

使用unordered_set 读写效率是最高的，并不需要对数据进行排序，而且还不要让数据重复

那有同学可能问了，遇到哈希问题我直接都用set不就得了，用什么数组啊。

直接使用set 不仅占用空间比数组大，而且速度要比数组慢，set把数值映射到key上都要做hash计算的。

不要小瞧 这个耗时，在数据量大的情况，差距是很明显的。

映射Map

java的Set和Map各有三个实现类

Set：HashSet、LinkedHashSet、TreeSet

Map：HashMap、LinkedHashMap、TreeMap

区别：

Set：

基于 性能 特点 使用场景
HashSet 哈希表 **O(1)**时间复杂度的插入、删除和查找操作 元素顺序可能会随插入顺序的变化而变化 当不关心元素顺序，仅需要快速访问元素时
LinkedHashSet 哈希表和链表 相比HashSet略微增加了内存开销，插入、删除和查找操作仍然是**O(1)**时间复杂度 保留元素的插入顺序 需要快速访问元素，同时保留插入顺序时
TreeSet 红黑树（平衡二叉树） **O(log n)**时间复杂度的插入、删除和查找操作 保证元素的排序，可以自定义顺序（因为是红黑树，可以前序中序后序） 需要对元素进行排序，且允许一定的性能开销时
共同点 元素通过add方法添加，删除用remove，查找用contains Set set = new HashSet<>(); 不允许重复元素

Map：

基于 性能 特点 使用场景
HashMap 哈希表 **O(1)**时间复杂度的插入、删除和查找操作 键值对的顺序可能会随插入顺序的变化而变化 当不关心键值对顺序，仅需要快速访问键值对时
LinkedHashMap 哈希表和链表 相比HashSMap略微增加了内存开销，插入、删除和查找操作仍然是**O(1)**时间复杂度 保留键值对的插入顺序或最近访问顺序 需要快速访问键值对，同时保留插入顺序时
TreeMap 红黑树 **O(log n)**时间复杂度的插入、删除和查找操作 保证键值对的排序,可以自定义排序（因为是红黑树，可以前序中序后序） 需要对键值对进行排序，且允许一定的性能开销时
共同点 键值对通过put方法添加，删除用remove，查找用get和containsKey Map<Integer, String> map = new HashMap<>(); 键唯一，不允许重复

C++中的Set和Map各有multi和unordered_：

multi可允许重复

unordered为无序（为什么要用？：需要键值对，但是不需要有序，因此无序更便捷）
